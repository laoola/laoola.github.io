## 一、学习路径总览

> **📌 核心要点**：每天1-1.5小时，8周建立AI工作流

```
第1周   → 了解工具（店小秘+九数云BI）
第2周   → 替代上架（批量刊登）
第3周   → 替代广告日报（BI配置）
第4周   → 预售/现货自动化
第5-8周 → 综合实战，补齐短板
第2-3月 → 智能化，提效翻倍
```

---

## 二、每周详细计划

### 第1周：了解工具

> **📌 目标**：找到最顺手的工具组合，不花钱先试

| 天 | 任务 | 耗时 | 产出 |
|----|------|------|------|
| 周一 | 注册店小秘，用免费版走一遍刊登流程 | 1小时 | 了解功能边界 |
| 周二 | 注册九数云BI，看官方教程走一遍 | 1小时 | 会建最简看板 |
| 周三 | 用DeepL翻译3个产品链接，对比质量 | 30分钟 | 判断AI翻译可用性 |
| 周四 | 注册实在Agent，看演示视频 | 30分钟 | 了解能做什么 |
| 周五 | 刷剪映AI官方教程，重点看"图生视频" | 1小时 | 了解视频AI能力 |
| 周末 | 休息，或整理笔记 | — | — |

**🚀 实践清单（完成后打勾）：**
- [ ] 店小秘走完刊登全流程
- [ ] 九数云BI建好第一个看板
- [ ] DeepL翻译3个链接，判断质量可用
- [ ] 实在Agent看演示，知道能做什么
- [ ] 剪映AI了解图生视频功能

**本周产出：** 7天结束能说出每个工具"能做什么、不能做什么"

---

### 第2周：批量刊登实战

> **📌 目标**：把77条上架从3小时缩到30分钟

| 天 | 任务 | 操作步骤 |
|----|------|----------|
| 周一 | 准备产品模板 | 在店小秘创建Excel模板，包含：标题/描述/价格/库存/属性 |
| 周二 | 批量导入测试 | 用模板导入10条，看成功率，排查失败原因 |
| 周三 | 图片批量上传 | 按SKU命名图片，批量上传到店小秘图片空间 |
| 周四 | 全流程测试 | 从导入到上架，走完10条全流程，记录耗时 |
| 周五 | 对比效率 | 手动上10条 vs 店小秘上10条，记录时间差 |
| 周末 | 复盘 | 找到瓶颈点，是模板问题还是操作问题 |

**🚀 实践清单（完成后打勾）：**
- [ ] 店小秘创建产品模板
- [ ] 批量导入50条链接
- [ ] 对比效率：手动 vs AI

**成功标准：** 50条链接在1小时内完成刊登
**失败应对：** 如果模板总是失败，换一种格式；或联系店小秘客服

---

### 第3周：广告日报自动化

> **📌 目标**：早上5分钟看完昨日广告数据，不用手动统计

| 天 | 任务 | 操作步骤 |
|----|------|----------|
| 周一 | 接入第一个广告平台 | 九数云BI接入TikTok广告数据源 |
| 周二 | 创建第一个看板 | 按花费/ROAS/曝光建最简看板 |
| 周三 | 添加告警 | 设置"花费比>目标值20%"触发企业微信通知 |
| 周四 | 对比人工统计 | 手动算昨天数据 vs AI看板数据，验证准确性 |
| 周五 | 接入第二个平台 | 接入亚马逊/Temu第二个广告平台 |
| 周末 | 复盘 | AI日报覆盖了哪些原来人工统计的指标？遗漏了什么？ |

**🚀 实践清单（完成后打勾）：**
- [ ] 九数云BI接入一个广告平台
- [ ] 创建第一个看板
- [ ] 设置异常告警
- [ ] 验证AI日报准确性

**成功标准：** 早上到公司，点开看板，5分钟看完所有核心指标
**失败应对：** 如果数据接入失败，看九数云官方文档排查，或换用Excel导入

---

### 第4周：预售/现货自动化

> **📌 目标**：库存规则自动匹配，人工只处理异常

| 天 | 任务 | 操作步骤 |
|----|------|----------|
| 周一 | 梳理现有规则 | 列出当前预售/现货的判断逻辑（如：库存<10为预售） |
| 周二 | 在店小秘配置规则 | 用店小秘的"库存规则"功能，设置自动切换条件 |
| 周三 | 测试3个SKU | 手动改一个库存，看系统是否自动切换 |
| 周四 | 跑一天观察 | 让系统自动运行一天，收集异常case |
| 周五 | 处理异常 | 人工处理4个星期积累的异常情况，完善规则 |
| 周末 | 复盘 | 自动切换成功率多少？需要人工介入的比例？ |

**🚀 实践清单（完成后打勾）：**
- [ ] 梳理预售/现货判断逻辑
- [ ] 在店小秘配置自动规则
- [ ] 运行1周观察效果

**成功标准：** 80%以上切换由系统自动完成
**失败应对：** 如果规则太复杂，用实在Agent代替店小秘处理

---

### 第5-6周：链接优化AI化

> **📌 目标**：7条链接优化从2小时缩到30分钟

| 天 | 任务 | 操作步骤 |
|----|------|----------|
| 周一 | 收集优化素材 | 整理7条链接现状（标题/主图/描述/属性） |
| 周二 | 用DeepL优化标题 | 批量英译中→中译英，看表达是否有改善 |
| 周三 | 用通义万相生成主图 | 选3条链接，用AI生成新主图备选 |
| 周四 | AB测试 | 原主图 vs AI主图，看点击率差异 |
| 周五 | 批量处理 | 把优化方法固化到工作流，不用每次重新想 |
| 周末 | 复盘 | AI优化的链接，点击率是否比原来高？ |

**🚀 实践清单（完成后打勾）：**
- [ ] DeepL优化7条链接标题
- [ ] AI生成3条链接主图
- [ ] AB测试对比点击率

**成功标准：** 优化一条链接从20分钟缩到5分钟（含人工审核）
**失败应对：** 如果AI图质量不行，保持人工制图，只用DeepL优化文案

### 第7-8周：早间数据处理自动化

> **📌 目标**：早上1.5小时缩到20分钟

| 天 | 任务 | 操作步骤 |
|----|------|----------|
| 周一 | 梳理早间流程 | 列出早上做的所有事情（订单/缺货/填数据/扫低利润单） |
| 周二 | 店小蜜处理订单 | 配置订单自动处理规则（正常单自动通过） |
| 周三 | 缺货自动登记 | 用九数云或店小秘设置缺货自动标记 |
| 周四 | 低利润单告警 | 九数云设置利润阈值，低于5%自动推送 |
| 周五 | 串联流程 | 把以上流程串起来，形成标准操作 SOP |
| 周末 | 复盘 | 早上实际花了多少时间？比原来快多少？ |

**🚀 实践清单（完成后打勾）：**
- [ ] 梳理早间流程清单
- [ ] 配置订单自动处理
- [ ] 设置低利润单告警
- [ ] 形成早间SOP

**成功标准：** 早间流程从1.5小时缩到20分钟
**失败应对：** 如果某个环节自动化失败，保留人工处理步骤作为兜底

---

### 第2-3月：智能化进阶

> **📌 目标**：从"人找问题"到"问题找人"

#### 投放智能化
- 开通 Perpetua（亚马逊广告）或平台官方AI投放
- 设置自动出价，让系统24小时优化
- 人工从"操作者"变成"监督者"

#### 选品数据化
- 订阅 Jungle Scout 或 Helium 10 基础版
- 每周用数据选品，不再"拍脑袋"
- 建立自己的产品数据库

#### 素材量产化
- 每天用剪映AI产出10-20条视频
- 建立视频素材库，按产品分类
- 测试不同卖点的转化率

## 四、学习时间表

```

午休时：        30分钟实战练习
```

## 五、常见问题应对

| 问题 | 应对 |
|------|------|
| 工具太复杂学不会 | 先放弃复杂功能，用最基础的功能跑通全流程 |
| AI质量不行 | 人工审核后再上线，AI只做初稿 |
| 数据接不进去 | 联系工具客服，一般都有在线支持 |
| 工具要收费 | 先用免费版，功能够用就不升级 |
| 效果不明显 | 记录数据对比，用事实说话 |
| 公司不让用外部工具 | 把数据留在本地，用Excel+AI作为替代 |

- 形成自己的SOP文档，以后可以教别人
