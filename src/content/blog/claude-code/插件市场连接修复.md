# Claude Code 插件市场连接问题排查解决文档

## 问题现象

启动 Claude Code 执行 `/plugin` 命令时，所有插件市场克隆失败：
```
Failed to connect to github.com port 443 after 21000 ms: Could not connect to server
```

**但手动在终端执行 `git clone` GitHub 可以成功。**

---

## 根因分析

1. **网络环境**：本地通过 v2rayN 代理访问 GitHub，系统代理开启
2. **问题本质**：
   - 终端手动执行 git 时，继承了终端的代理环境变量 → 成功
   - Claude Code 启动时，没有代理环境变量 → git 直连 GitHub → 超时失败

---

## 排查步骤

| 步骤 | 操作 | 结论 |
|------|------|------|
| 1 | `ping github.com` | ICMP 可达，0% 丢包，说明网络连通 |
| 2 | `curl -I https://github.com` | HTTPS 连接超时，说明 443 端口无法直连 |
| 3 | `env | grep -i proxy` | 无代理环境变量 |
| 4 | `git config --global --list | grep -i proxy` | Git 无代理配置 |
| 5 | 查看 v2rayN 设置 | 本地混合监听端口 **10808** |

---

## 解决方案

### 1. 配置 Claude Code 代理环境变量

文件：`~/.claude/settings.json`

```json
{
  "env": {
    // ... 其他配置
    "HTTP_PROXY": "http://127.0.0.1:10808",
    "HTTPS_PROXY": "http://127.0.0.1:10808"
  }
}
```

### 2. 配置 Git 全局代理

```bash
git config --global http.proxy http://127.0.0.1:10808
git config --global https.proxy http://127.0.0.1:10808
```

### 3. 修复插件市场配置格式

每个市场条目必须包含 `lastUpdated` 字段：

```json
"affaan-m-everything-claude-code": {
  "source": {
    "source": "git",
    "url": "https://github.com/affaan-m/everything-claude-code.git"
  },
  "installLocation": "C:\\Users\\zwh\\.claude\\plugins\\marketplaces\\affaan-m-everything-claude-code",
  "lastUpdated": "2026-03-30T03:00:00.000Z",
  "autoUpdate": false
}
```

## 禁用/启用自动更新

**禁用自动更新**（每个市场添加）：
```json
"autoUpdate": false
```

**启用自动更新**（每个市场添加）：
```json
"autoUpdate": true
```
