# 安装新插件后斜杠命令没有自动补全提示

## 问题现象

安装新插件（如 `everything-claude-code`）后，输入 `/` 时，新安装的插件提供的命令没有出现在自动补全提示列表中。

## 根因分析

Claude Code 需要显式重新加载所有已安装插件，才能将新插件中的命令注册到自动补全系统。安装插件后不会自动触发重新加载。

## 解决方案

在 Claude Code 中运行以下命令：

```
/reload-plugins
```

这个命令会：
1. 重新读取所有已安装插件的定义
2. 注册所有新的命令、技能和代理
3. 更新斜杠自动补全缓存

## 验证

运行完 `/reload-plugins` 后，你会看到类似输出：

```
Reloaded: 3 plugins · 60 skills · 34 agents · 29 hooks · 6 plugin MCP servers · 0 plugin LSP servers
```

然后再次输入 `/`，就能看到所有新安装的命令出现在自动补全提示中了。

## 相关信息

- 这是 Claude Code 的标准工作流程，不是 bug
- 每次安装新插件后都需要这一步
- 如果还是不显示，可以尝试重启 Claude Code CLI

## 注意事项

### 1. 全局规则存储位置

**错误做法**：将全局规则放在 `~/.claude/memory/` 目录下
- memory 目录用于项目级和用户级自动记忆，不承载执行规则
- 规则不会被自动加载，Claude 无法读取遵守

**正确做法**：将全局规则放在 `~/.claude/rules/common/` 目录下
- 这个目录是 Claude Code 官方约定的全局规则加载路径
- 所有项目都会自动加载这里的 `.md` 规则文件
- 建议按主题拆分：`agents.md`、`code-review.md`、`security.md` 等

### 2. 插件启用配置

在 `~/.claude/settings.json` 的 `enabledPlugins` 中，需要将插件设置为 `true` 才能启用：

```json
{
  "enabledPlugins": {
    "everything-claude-code@affaan-m-everything-claude-code": true
  }
}
```

如果设置为 `false`，插件安装了也不会加载，命令不会出现在补全列表中。

### 3. MCP 服务器数量说明

everything-claude-code 默认提供 6 个 MCP 服务器：
- context7 - 文档查询
- exa - 网页搜索和爬取
- github - GitHub 操作
- memory - 知识图谱记忆
- playwright - 浏览器自动化
- sequential-thinking - 分步思考

**这是正常配置，不算过多**：
- MCP 服务器保持连接但闲置时消耗资源很少
- 工具是按需调用，不调用不执行
- 当前上下文使用率通常在 20-30%，有足够余量

## 使用建议

### 日常操作流程

1. **安装插件后**：立即运行 `/reload-plugins`
2. **检查加载结果**：确认输出中的数量符合预期
3. **验证补全**：输入 `/` 检查新命令是否出现
4. **清理不使用的插件**：在 `enabledPlugins` 中禁用不用的插件即可，不需要删除

### 性能优化建议

| 场景 | 建议 |
|------|------|
| 配置了很多插件但很少用 | 在 `settings.json` 禁用不用的，只保留常用 |
| 上下文使用率超过 70% | 重启 Claude Code CLI 刷新上下文 |
| 只需要 ECC 核心功能 | 不需要禁用任何 MCP，默认配置就是最优 |

### 常见问题

**Q: 显示 "Reloaded: 0 plugins" 是什么意思？**
A: 没有插件被启用，检查 `enabledPlugins` 配置是否正确。

**Q: 为什么输入斜杠后只显示部分命令？**
A: Claude Code 补全会根据上下文动态过滤，继续输入命令前缀会过滤出更精确的结果。

**Q: 我有 10+ 个启用的插件，这会很慢吗？**
A: 影响不大，主要影响是启动时的一点延迟，运行中影响很小。

**Q: 技能激活之后怎么关闭？**
A:

| 技能类型 | 关闭方法 |
|----------|----------|
| 单次执行技能 | 执行完自动结束，不需要手动关闭 |
| `/loop` 循环技能 | 使用 `/stop` 命令停止循环 |
| 钩子 Hooks | 需要在 `~/.claude/settings.json` 中删除或禁用配置，重启生效 |
| 持续代理任务 | 使用 `/stop` 命令可以终止正在运行的代理任务 |

大多数技能都是单次调用，运行完就完成了，不会持续占用资源。只有循环和定时任务需要显式停止。
